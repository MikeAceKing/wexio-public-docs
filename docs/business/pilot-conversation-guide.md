# Pilot conversation guide

Updated: 2026-07-04

This guide helps explain Wexio to a company during a pilot discussion.

## Opening message

Wexio is not presented as one more isolated application. It is a business workspace where a company can bring users, documents, modules, and workflows together.

The goal of a pilot is to discover where the company loses time today and test whether Wexio can create a clearer way to work.

## Questions to ask

### Current tools

- Which tools do you use every day?
- Which information is duplicated?
- Where do users lose time?
- Which files are hard to find?
- Which process still depends on manual follow-up?

### Users

- Who should use Wexio first?
- Who needs admin access?
- Who only needs a limited screen?
- Are any external users involved?

### Modules

- Do you need clients or CRM first?
- Do you need document organisation first?
- Do you need time tracking or HR first?
- Do you need invoice preparation or finance visibility first?
- Do you need projects, workflow, or support first?

### Success criteria

- What should be easier after two weeks?
- Which repeated task should take less time?
- Which information should be easier to find?
- Which module would prove the most value?

## How to explain the status

Use clear language:

Wexio is ready for selected controlled pilot demonstrations. The pilot is used to validate scope, users, modules, and value before wider rollout.

## How to avoid overpromising

Do not present every module as active for every company. Do not present external provider workflows as proven before a dedicated setup and proof.
