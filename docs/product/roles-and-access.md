# Roles and access

Updated: 2026-07-04

Wexio is role-aware. A company workspace should not show the same tools and information to every user.

## Why roles matter

A business workspace can include administrators, employees, workers, clients, guests, and specialist profiles. Each group needs a different experience.

The purpose of roles is to keep work clear:

- administrators manage the workspace;
- employees work inside internal modules;
- workers use worker-focused screens;
- clients use limited customer-facing screens when enabled;
- guests or viewers receive restricted access only.

## DesktopShell users

Internal workspace users can use the desktop-style workspace. Their widgets, shortcuts, layout, and active modules can be personal to them.

## Worker and client users

Worker and client users should not receive the full internal workspace by default. They can use dedicated surfaces designed for their scope.

## Module visibility

A saved shortcut or visible label is not enough to grant access. The module still needs to be enabled and allowed for the current user.

## Practical pilot questions

Before a pilot starts, the company should define:

- who are the first users;
- what each user type should see;
- which modules each group needs;
- what should stay admin-only;
- which external users, if any, are involved.

## Current boundary

Role behavior is part of the controlled pilot validation. Wexio should be demonstrated with real user profiles and realistic business tasks.
